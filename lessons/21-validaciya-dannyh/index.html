🚀 1. Введение

Тема урока — валидация данных в формах React-приложения. Сравниваем два главных инструмента:

⚡ Zod

  • 20M скачиваний/нед
  • TypeScript-first
  • Популярен на бэке (Nest.js, tRPC)
  • Мощная документация

🟡 Yup

  • 8M скачиваний/нед
  • Проще синтаксис
  • Исторически популярен
  • Хорошая документация на сайте

Принцип работы одинаков: создаём схему → передаём данные → получаем успех или ошибки.

⚙️ 2. Подготовка окружения
  • Убедиться: React, TypeScript, MUI, Redux Toolkit установлены
  • Установить Zod: npm install zod
  • Установить Yup: npm install yup
  • React Hook Form: npm install react-hook-form
  • Resolver для Yup: npm install @hookform/resolvers
✋ 3. Ручная валидация

Пример в Login.tsx — проверяем email вручную:

  • Ищем @ и проверяем текст слева/справа
  • Показываем ошибку через notistack
  • Подсвечиваем поле через error в MUI TextField

Проблемы подхода

  • Код раздувается для сложных проверок (домен, длина, спецсимволы)
  • Нет типизации ошибок — легко ошибиться
  • Плохо масштабируется для форм с множеством полей
⚡ 4. Валидация с Zod

Основы

  • Создаём схему: z.string().email(), z.string().min(8)
  • parse — проверка данных, ошибки через try/catch
  • Ошибки: ZodError с массивом объектов (код, сообщение, путь)

Пример схемы

import { z } from 'zod';

const loginSchema = z.object({
  email: z.string().email('Неверный формат email'),
  password: z.string().min(8, 'Минимум 8 символов'),
});

type LoginForm = z.infer<typeof loginSchema>;

Преимущества

  • Строгая типизация с TypeScript
  • Сложные проверки: regex, кастомные сообщения
  • Популярен на бэкенде и фронтенде
🟡 5. Валидация с Yup

Основы

  • Схема: object({ email: string().email(), password: string().min(8) })
  • validate / validateSync — проверка данных
  • abortEarly: false — собрать все ошибки, а не только первую

Пример схемы

import * as yup from 'yup';

const loginSchema = yup.object({
  email: yup.string().email('Неверный формат').required(),
  password: yup.string().min(8, 'Минимум 8 символов').required(),
});

Преимущества и недостатки

  • ✅ Проще синтаксис, меньше бойлерплейта
  • ✅ Хорошая документация на сайте
  • ❌ Меньше скачиваний (уступает Zod 2.5x)
  • ❌ TypeScript-интеграция чуть слабее
🪝 6. React Hook Form

Почему React Hook Form?

  • 8M скачиваний — в 10x популярнее Formik
  • Нет лишних ре-рендеров, мемоизированные хуки
  • Меньше бойлерплейта

Интеграция с Zod

import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';

const { register, handleSubmit, formState: { errors } } = useForm({
  resolver: zodResolver(loginSchema),
});

// В JSX:
<TextField
  {...register('email')}
  error={!!errors.email}
  helperText={errors.email?.message}
/>

Режимы валидации

  • mode: 'onSubmit' — по умолчанию
  • mode: 'onChange' — мгновенная валидация при вводе
  • mode: 'onBlur' — после потери фокуса
  • mode: 'onTouched' — после первого касания
📦 7. Кратко о Formik
  • Компонент <Formik> с initialValues, validationSchema, onSubmit
  • Внутри используем handleChange, handleBlur, handleSubmit

Почему не использовать

  • Больше бойлерплейта и вложенности
  • Лишние ре-рендеры, сложнее мемоизировать
  • TypeScript-интеграция хуже
  • 800k скачиваний vs 8M у React Hook Form

Когда использовать

  • Если проект уже на Formik — переписывать нецелесообразно
  • Использовать useFormik (хуковый API) вместо <Formik>
📝 8. Домашнее задание
1
Форма логина с Zod + React Hook Form: email обязательный, password мин. 8 символов + хотя бы 1 спецсимвол (regex)
2
Переписать ту же форму с Yup — сравнить синтаксис
3
Форма регистрации: проверка совпадения пароля и подтверждения (.ref в Yup или .refine в Zod)
★
Опционально: кастомные сообщения об ошибках, режим onBlur, исследовать Joy (11M скачиваний)
